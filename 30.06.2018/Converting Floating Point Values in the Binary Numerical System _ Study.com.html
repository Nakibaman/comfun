This lesson talks about floating point binary number representation and explains how the conversion in both directions is done, along with its advantages and disadvantages.

Introduction

Floating point denotes those numbers which are composed of digits that may or may not be fixed before and after the decimal point, that is, the decimal can float and the number can be written in different formats. Since the computer converts all the data to be processed in the binary bit format, we will understand how such conversions can be carried out through the course of this lesson. Floating point numbers can be categorized into two types :

  • Single precision floating point: it is 32-bit floating point number, that is, it is a number that occupies 32 bits of computer memory
  • Double precision floating point: it is 64-bit floating point number, that is, it is a number that occupies 64 bits of computer memory

In order to understand better the conversion of floating point values to binary numbers, lets understand the number system.

Number System in Computer Science

Number system in Computer Science comprises different types of representation of a given number, namely binary, decimal, hexadecimal and octal.

  • Binary: Numbers that have base 2 and are represented by two digits, 0 and 1.
  • Decimal: Numbers that have base 10 and are represented in the general format that is followed to represent numbers.
  • Hexadecimal: These have base 16 and a number can be represented by sixteen symbols in this format.
  • Octal: These have base 8 and a number can be represented by eight symbols in this format.

Binary Fraction Representation

Binary fractional numbers in Computer Science can be either a fixed point value number or a floating point value.

  • Fixed-point values: These types of numbers consist of fixed digits before and after the radix/decimal point. These are used to represent fractions in base 2 or 10 formats.
  • Floating point values: These types of fractions are also represented in base 2 and base 10 formats, the only difference is that the decimal point can float, or be represented in different positions.

Representing a Floating Point Value

The floating point numbers are broken down into sign, mantissa, and exponent bits when representing it in the binary format. For example, if we take a floating point number -4.124*103, we can say that:

The sign bit is 1 (which represents the negative sign of the number; in case it is positive, the value of the sign bit is 0). This is a single bit value.

The mantissa is the actual number, that is, 4.124. The portion after the decimal point in the mantissa is obtained by adding each digit of the number by multiplying each digit by a power of 10. That is, 1/10 + 2/100 + 4/1000. It is a 23-bit value.

The exponent represents the exponential power which is 3 in this case. This is an 8-bit value.

Conversion of Floating Point Value in Binary System

Converting a floating point number to decimal involves the following steps:

  • Separately process the decimal and binary parts of the number and convert them into binary format
  • Normalize the binary number by moving the decimal point to the leftmost position
  • Convert the exponent part into binary

For example, let us take a fractional number 36.890625 and convert it to binary form.

Step 1 - Convert the integer part of the number to binary

Fig 1 Integer part to binary

After converting 36 into binary, we get the result as 100100

Step 2 - Convert the mantissa part to binary

  • Multiply the exponent part by 2
  • If the resulting product is a whole number, that is >=1.0, the binary value is taken as 1, else it is 0
  • Once the product reaches 1.0, the multiplication is stopped and the resultant binary values are taken
  • The mantissa is of 23 bits, so the conversion is terminated as soon as 23 bits are attained

Fig 2 Final step to binary conversion

Thus, the binary conversion of 36.890625 is given as 100100.111001*20.

Step 3 - Normalize the exponent

In this step, the decimal point is shifted to the left-most position and the exponent is adjusted. In the above case, the decimal point needs to be shifted by 5 points resulting in 1.00100111001*25. The value of the adjusted exponent is 5.

A binary floating point exponent value has a positive and a negative range. Since exponent is an 8-bit value which can have 256 different values, it is divided into positive and negative range wherein for a positive or negative range, 127 is added (or reduced) to the exponent value. In this case, we have a positive exponent value. Thus it will be 5+127=132 which when converted to binary gives 10000100.

Step 4 - Final binary value of the floating point number

First 0 is the sign value which is 0 for positive number and 1 for negative; the mantissa includes digits after the decimal point and the 1 before the decimal point is ignored; the exponent value is written as it is.

Pros and cons of floating point representations

Pros of floating point representation:

  • Greater range of numbers can be represented
  • It has a constant relative error
  • Floating point algorithms are easier to design and debug

Cons of floating point representations:

  • More storage is required
  • Slower processing times
  • There is a lack of precision, i.e., some real numbers can be represented in approximations only

Lesson summary

Number system in computer science is categorized as binary, octal, decimal and hexadecimal, and the computer converts each and every data into binary before commencing any operation. For converting a floating point number from decimal format to binary involves a step-by-step process. First, the integer part of the number is converted to binary. Next, the mantissa part, that is the part after the decimal point, is converted to binary by multiplying the exponent by 2 and until we get a 23-bit mantissa in the binary format. Last step is to normalize the exponent and put together the integer and mantissa parts, and determine the value of the sign bit. The sign bit is 1 if the number is negative, and 0 if it is positive.
